# M2 — Claim modeling (deferred)

## Goal

- **Audit #4:** stop using OIDC `profile` scope as a transport for app-specific claims (`isAdmin`, `chapter`, `chapters`). Move them to a custom scope `gdgjp:chapters`.
- **Audit #5:** stop denormalizing `isAdmin` onto every RP's `user` table. Read it through `getFreshClaims`.
- **Audit #6 (additive only):** add `idp_sub` to each RP's `user` table; populate from `account.accountId` via a post-callback hook. FK rekey on `links.owner_user_id`/`images.user_id`/`events.owner_user_id` is **not** in scope.

## Files

IdP:
- `accounts/app/lib/auth.server.ts` — register `gdgjp:chapters` via `oidcProvider({ scopes: [...] })`. Change `getAdditionalUserInfoClaim` to gate on `scopes.includes("gdgjp:chapters")` instead of `"profile"`.

Library:
- `gdg-lib/src/auth/server.ts` — RP `genericOAuth` `scopes` list adds `"gdgjp:chapters"`. Remove `isAdmin` from `mapProfileToUser` and from `additionalFields` (user table column goes away). Add a post-link hook that copies `account.accountId` → `user.idp_sub`.
- `gdg-lib/src/auth/index.ts::AuthUser` — drop `isAdmin` field, or mark as derived from claims only.

Per-RP migrations:
- `tinyurl/migrations/00XX_user_idp_sub_and_drop_isadmin.sql`
- `img/migrations/000X_user_idp_sub_and_drop_isadmin.sql`
- `scheduler/migrations/000X_user_idp_sub_and_drop_isadmin.sql`

Each: `ALTER TABLE "user" DROP COLUMN "isAdmin"; ALTER TABLE "user" ADD COLUMN idp_sub TEXT; CREATE UNIQUE INDEX user_idp_sub_idx ON "user"(idp_sub) WHERE idp_sub IS NOT NULL;`

## Open design questions for M2

- Should the custom scope require user consent (different from `skipConsent: true` trusted clients)? Probably no — first-party.
- Backfill strategy for `idp_sub` on existing rows: on next login the hook will populate it; rows whose user hasn't re-logged in stay NULL. Acceptable for an additive column.

## Verification (when M2 ships)

- Local: sign in fresh; confirm `user.idp_sub` is populated, `user.isAdmin` column is gone, and chapter info no longer rides on the `profile` scope.
- IdP `.well-known/openid-configuration` → `scopes_supported` contains `"gdgjp:chapters"`.
- RPs request the new scope (visible in the OAuth authorize URL during sign-in).
