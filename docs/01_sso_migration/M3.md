# M3 — RP-Initiated Logout (deferred)

## Goal

- **Audit #3:** replace iframe-based federated logout with OIDC RP-Initiated Logout (top-level redirect chain via each RP's `frontchannel_logout_uri`). No third-party cookie reliance.

## Approach

Two viable shapes:

1. **Sequential top-level chain.** IdP redirects to RP1's `frontchannel_logout_uri`, which clears the RP1 cookie and redirects back to the IdP with a continuation token. IdP redirects to RP2's URI, and so on. Final hop lands at `post_logout_redirect_uri`. Slow but maximally compatible.

2. **Visible "Signing out…" page with parallel top-level navigations** (one per RP) via `window.open(..., '_self')` chained or sequential `location.replace` driven by JS. Bit hackier; could work if (1) is too slow with 4+ RPs.

Pick (1) first; fall back to (2) if UX is too sluggish.

After M3, the RP `auth/signout` can redirect straight to the IdP's `end_session_endpoint` (because the IdP-side handler now triggers the chain itself), letting us delete `handleSignOutIframe` and the `auth.signout-iframe.ts` routes entirely.

## Files

Library:
- `gdg-lib/src/auth/server.ts::handleFederatedSignOut` — replace iframe renderer with redirect-chain orchestration.
- `gdg-lib/src/auth/server.ts::IdpClient` — add `frontchannelLogoutUri?: string`.

Each RP:
- Rename `auth.signout-iframe.ts` → `auth.frontchannel-logout.ts`. The handler verifies `iss` query param matches `IDP_URL`, clears the local session cookie, then redirects back to the IdP's continuation URL.

IdP:
- `accounts/app/lib/auth.server.ts::trustedClientsFromEnv` — add `frontchannelLogoutUri` per RP (env-derived).
- `accounts/app/routes/auth.signout.ts` — switch from `handleFederatedSignOut`'s iframe variant to the new chain variant.

Wrangler:
- `accounts/wrangler.toml` (and each RP's) — new env: `*_FRONTCHANNEL_LOGOUT_URL`.

Discovery:
- Confirm `frontchannel_logout_supported: true` appears in `.well-known/openid-configuration`. better-auth advertises this automatically when `oidcProvider({ frontchannelLogoutSupported: true })` is set — verify.

## Verification (when M3 ships)

- In a fresh browser profile with third-party cookies blocked, sign out at the IdP → confirm every RP cookie is cleared after the chain.
- Mid-chain interruption (close tab) → confirm remaining RPs are cleared on next request via short cookie TTL OR (if M4 is also live) via cached-claims invalidation.
